## 是什么

本地跑的**防水卷材国标问答库**：导入国标/行标 PDF → OCR → 指标行切块 → 向量化入库；提问时 Agent 混合检索、标来源作答，库内查不到走联网兜底。

技术栈：**Mastra(TS) + libSQL + OpenRouter + PaddleOCR-VL + Tavily**；前端 **Vite + React + ai-elements**。

## 三条流

1. **入库（离线）**：`pdf/*.pdf` → PaddleOCR-VL（结果缓存，可反复重切不重 OCR）→ 指标行切块（ADR-0004）→ embedMany → libSQL `upsert`。
2. **问答（在线）**：前端 `useChat` → `/api/chat` → `standardsAgent`（`hybridQueryTool` 向量+BM25+元数据过滤 / 服务端 threadId 记忆）→ AI SDK UI-message 流 → 前端 ai-elements 渲染（ADR-0006）。
3. **联网兜底**：库内查不到（含未入库标准与库外问题）→ `webSearchTool`（Tavily）→ 结果标「来源：联网」。

## Web 应用

- **前端** `ui/`（专业暗色三栏工作台）：登录 / 入库 / 检索台，React Router 路由。
- **鉴权**：单 admin、`.env` 凭据、httpOnly 签名 cookie（ADR-0007）。
- **后端** `src/web.ts`：`/api/chat|ingest|library|threads|messages` + 托管 `ui/dist`。

## 关键约定（详见对应 ADR）

- 指标行切块 + 混合检索 + 元数据过滤（[ADR-0004](adr/0004-indicator-chunking-hybrid-retrieval.md)）。
- embedding 锁 `text-embedding-3-small`，入库与检索同模型；对话 `deepseek/deepseek-v4-flash`，全收口 OpenRouter（[ADR-0001](adr/0001-model-routing-split.md)）。
- OCR 走 PaddleOCR-VL 托管 API，不本地渲染、不引 Python（[ADR-0003](adr/0003-ocr-paddleocr-vl.md)）。
- 废止标准当普通文档处理（[ADR-0005](adr/0005-deprecated-as-normal.md)）。
