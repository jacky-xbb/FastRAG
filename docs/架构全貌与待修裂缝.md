# 架构全貌与待修裂缝

> 记录日期：2026-06-15。本文是一次「读代码摸全貌」会话的沉淀，既给人看，也给未来的 AI 助手当上下文。

## 1. 这个项目是什么

本地跑的**防水卷材国标问答库**：国标 PDF → OCR → 切块 → 向量化入库；提问时 Agent 检索知识库、用 gpt-5 作答、标来源，查不到就联网兜底。（废止标准当普通文档处理，见 ADR-0005）

技术栈：**Mastra(TS) + libSQL + OpenRouter + PaddleOCR-VL + Tavily**。

## 2. 三条主线（整个系统就这三条管道）

### 流① 入库（离线，跑一次）
```
pdf/*.pdf
 └→ ocr.ts    调 PaddleOCR-VL 托管 API → 缓存 ocr_cache/*.json
 └→ ingest.ts 读 JSON → chunk.ts 切块 → config.ts 拿 embedding 模型
              → embedMany(text-embedding-3-small) → LibSQLVector.upsert
 → standards 表（859 块 + 1536 维向量）
```
关键：`ocr.ts` 与 `ingest.ts` **不直接互调**，靠 `ocr_cache/*.json` 文件接力。所以可一次 OCR、反复改切块重 ingest，不重跑花钱的 OCR。

### 流② 问答（在线，每次提问）
```
网页 → server.ts(/api/ask) → agent.ts ask() → mastra.ts 的 Agent
   → Agent 自己决定调 ragTool(retrieve.ts) → 查 standards 表
   → gpt-5 生成答案 + 写会话历史(mastra_threads/messages)
```

### 流③ 联网兜底
```
网页 → server.ts(/api/web) → web.ts → Tavily
```

## 3. 文件地图（src/ 10 个文件）

| 文件 | 角色 |
|---|---|
| `config.ts` | 模型出口：openrouter(对话) + fniju(embedding)。**公共水龙头**，入库与检索都从它取同一个 embedding 模型 |
| `mastra.ts` | **总装配/心脏**：Agent + 向量 + 存储 + 日志 + ragTool |
| `ocr.ts` | PDF→JSON（PaddleOCR-VL），不被任何文件 import，单独跑 |
| `chunk.ts` | JSON→指标行切块；文件名带「作废」→ 状态=废止 |
| `ingest.ts` | 切块→向量→入库 |
| `retrieve.ts` | `ragTool`：向量 + 元数据过滤检索 |
| `agent.ts` | 问答入口 + CLI（「按启动键的人」） |
| `server.ts` | HTTP 接口 /api/ask、/api/web（总入口/传达室） |
| `web.ts` | Tavily 联网搜索 |
| `db.ts` | 裸 libSQL 客户端，目前没人正经用（留给将来全文检索） |

### 依赖关系（A → B = A 用 B）
```
config.ts ← ingest.ts ← chunk.ts
config.ts ← retrieve.ts → mastra.ts ← agent.ts ← server.ts → web.ts
ocr.ts、db.ts = 当前游离
```

## 4. 数据落地（vector.db）

| 表 | 行数 | 作用 |
|---|---|---|
| `standards` | 859 | 命根子：指标行切块 + 向量 + metadata |
| `standards_vector_idx_shadow` | 859 | 向量索引影子表，libSQL 自动维护 |
| `libsql_vector_meta_shadow` | 1 | 索引配置（维度=1536） |
| `mastra_threads` / `mastra_messages` | 各 13 | 会话历史 |
| 其余 ~35 张 `mastra_*` | 0 | Mastra 框架预建空表，别删 |

每行 `embedding` 是 6144 字节（1536 维 × 4）。GUI 显示 `EMPTY`/`id=NULL` 是显示层问题（BLOB 不渲染、SERIAL 在 libSQL 不自增），数据本身正常。

## 5. ⚠️ 三道「文档说了、代码没做到」的裂缝

1. **检索退化成纯向量**（`retrieve.ts:43`）：BM25+RRF 混合被注释成 `hybridSearch_DEPRECATED`，违反 CLAUDE.md 硬约束 2（检索 = 向量 + BM25 + 元数据过滤）。**影响最大，建议先修。**
2. **部分表没按指标行切块**：GB/T 18242-2025 整张表落进正文块（表名/指标名为空），违反硬约束 1。根因在 `chunk.ts` 只处理 `raw.tables`，而那份 PDF 的 OCR 没产出结构化 `tables`。
3. **一份 PDF 没入库**：`pdf/JTST 232-2019 水运工程材料试验规程.pdf` 在目录里但不在 `standards` 表（库内 17 个文档，目录 18 个）。
