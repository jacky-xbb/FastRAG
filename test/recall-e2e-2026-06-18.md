# 端到端召回率评测报告 — 2026-06-18（老数据集）

- **库**：本地种子库 `file:./vector.db`（1865 块，今日重灌）。与 Fly 卷 `/data/vector.db` **同源**——Fly 库即由此本地种子库 deploy 烤入，故本地跑 = 测线上用户打到的同一份标准语料。
- **两种口径对照**：
  - **直连 P0（裸召回）**：`test/eval.ts` 直接调 `hybridSearch()`，Recall@6。量检索器自身的召回底线，**不经 Agent**（回放）。
  - **端到端召回（`--e2e`）**：每题灌进真实 Agent（`agent.stream`，与线上 `/api/chat` 同一条路），抓 Agent **实际**调 `hybridQueryTool` 拉回上下文的块文本判命中。Agent 自定 topK / 自抽 standardCode / 可多轮检索，故判**二值命中**（非 @rank）。这才反映真实使用。
  - 端到端同一次跑顺带得 **P2 来源正确率**（答案是否引到对的标准号）。
- 本次只覆盖**老数据集**（指标 40 + 正文 32）。新数据集（指标 16 / 正文 20）改在 Fly 上跑，另出报告。

## 一句话结论

老数据集端到端召回（单跑）：**指标 97.5%（39/40）、正文 90.6%（29/32）**；来源正确率**指标 97.5% / 正文 100%**。

关键发现：**端到端正文召回反而高于直连裸召回（84.4%）**——真实 Agent 的 query 改写 + 多轮检索，把直连一把召不回的「无号正文」题全捞了回来（无号 20/20 = 100%）。

> ⚠️ **单跑 e2e 对边界题有抖动**：e2e 每题只跑一次，而 Agent 每次自拟检索词不同，结果有运行间方差。复测发现 #9（带号正文）实为 2/3 命中，本次单跑恰好撞上坏路径被判 ✗。故「带号正文 75%」是单跑下界，稳定值更接近 83%（10/12）。要出可发布数字，应每题跑 N 次取多数/最好（见末尾「方法论待办」）。

## 逐题差异分析

### 指标 40 题：端到端 = 直连（无差异）
两口径同召回 39/40，唯一未召回同一题：

- **#15**（无号）「自粘聚合物改性沥青防水卷材的钉杆撕裂强度要求多少？」——直连、端到端均未召回；端到端答案来源也因此判错/缺。已知薄弱：长产品名 + 冷门指标（钉杆撕裂）+ 无号。

> 指标题答案在指标表行，锚点强、BM25/向量都好命中，Agent 决策几乎不带来增减——故两口径一致。

### 正文 32 题：端到端净 +2（捞回 3、退化 1）

**端到端捞回（直连✗ → 端到端✓），全为「无号」题**，靠 Agent 改写/多轮检索补上：

| # | 题 | 直连 | 端到端 |
|---|---|---|---|
| 12 | SBS 改性沥青防水卷材以什么为主要原料？（无号） | ✗ | ✓ |
| 25 | 铁路桥面防水层按材料分成哪些类型？（无号） | ✗ | ✓ |
| 26 | 铁路隧道防水板适用于多高时速的工程？（无号） | ✗ | ✓ |

**端到端抖动（直连✓ → 端到端单跑✗，但复测 2/3 命中）**：

| # | 题 | 直连 | 端到端单跑 | 复测 | 实质 |
|---|---|---|---|---|---|
| 9 | GB/T 23457 预铺防水卷材适用于与什么粘结？（带号） | ✓@2 | ✗ | **2/3 命中** | 抖动非退化 |

复测 3 次：2 次 Agent 一把检索就命中第5页「范围」块（自粘胶/后浇混凝土/粘结 三词齐）；1 次 Agent 连发 7 次检索越改越偏，还自加 `page:1` 过滤（范围实在第5页，被滤掉），全召回成表3/表4 剥离强度块而漏词。**线上手测该题答案与来源均正确**。教训：①单跑 e2e 有方差；②Agent 偶发过度多轮改写 + 自作主张加 `page` 过滤会挤掉「范围」类正文块——间歇、非硬伤。

**两口径均未召回（硬骨头）**：

- **#7**（带号）「GB/T 18242 …以什么为主要原料？胎基是什么？」——原料/胎基在前言或范围段，正文切块未把三要素聚到同一块。
- **#14**（带号）「JTS/T 232 焊接接头拉伸试验测定什么？判断什么？」——JTS/T 232 是《水运工程材料试验规程》，含钢筋/焊接等宽范围内容，目标段落召回脆弱。

## 复现命令

```bash
# 直连裸召回（快）
npx tsx test/eval.ts            # 指标 40
npx tsx test/eval.ts --prose    # 正文 32

# 端到端召回（跑真实 Agent，慢，含 P2）
npx tsx test/eval.ts --e2e
npx tsx test/eval.ts --e2e --prose

# 辅助：--limit N 只跑前 N 题（小样验证 / 控成本）
npx tsx test/eval.ts --e2e --limit 3
```
